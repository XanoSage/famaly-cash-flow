# Localization

## Цель

Сервис должен быть удобен двум пользователям семьи:

- русский язык для основного пользователя;
- украинский язык для жены.

## MVP Requirements

- Язык выбирается на уровне пользователя.
- Поддерживаемые языки MVP: `ru`, `uk`.
- Интерфейс личного кабинета должен иметь переводы на оба языка.
- Системные категории должны иметь названия на обоих языках.
- Отчеты и инсайты должны генерироваться на языке текущего пользователя.
- Bank raw values сохраняются как есть, без перевода.

## Что переводим

- Навигация.
- Названия экранов.
- Кнопки.
- Фильтры.
- Статусы импорта.
- Ошибки и уведомления.
- Названия системных категорий.
- Тексты инсайтов.
- Периоды и форматированные подписи.

## Что не переводим

- Raw description из банка.
- Raw bank category.
- Merchant name.
- Комментарии пользователя.
- Имена файлов.

## Технический подход

Frontend:

- Использовать i18n-словарь для React.
- Хранить ключи, а не текст прямо в компонентах.
- Язык брать из профиля пользователя.

Backend:

- Хранить системные категории с translation keys.
- Для API возвращать либо translation key, либо локализованное имя по `Accept-Language` или настройке пользователя.
- Инсайты генерировать через шаблоны на `ru` и `uk`.

## Открытые вопросы

- Нужно ли на первом экране логина переключать язык до входа?
- Какой язык использовать по умолчанию для нового пользователя?
- Нужны ли украинские названия для пользовательских категорий, созданных на русском, или они остаются как введены?
