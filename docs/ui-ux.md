# UI/UX

## Формат

MVP - веб-приложение с личным кабинетом.

Frontend stack:

- React;
- Vite;
- TypeScript.

Интерфейс должен быть адаптивным:

- desktop удобен для анализа, таблиц и импорта XLSX;
- mobile удобен для просмотра и быстрого ручного ввода.

Интерфейс должен поддерживать русский и украинский языки. Язык выбирается в настройках пользователя.

## Разделы личного кабинета

### Dashboard

- Сводка за период.
- График расходов.
- Расходы по категориям.
- Топ продавцов.
- Блок накоплений.
- Блок бюджета: общий лимит, использовано, важные категории.
- Блок рабочих/FOP операций отдельно от семейной аналитики.
- Простые инсайты.

### Import

- Загрузка XLSX.
- Preview операций.
- Подсветка дублей.
- Редактирование категории перед импортом.
- Подтверждение импорта.

### Transactions

- Таблица операций.
- Фильтры.
- Поиск.
- Редактирование операции.
- Ручное добавление операции.
- Фильтр по физической/виртуальной карте как payment instrument.
- Очередь операций "на проверку" для переводов физлицам и платежей по реквизитам.
- Фильтр по scope: семейные, личные, рабочие/FOP.

### Categories

- Список категорий и подкатегорий.
- Создание своих категорий.
- Управление правилами.

### Merchants

- Список продавцов.
- Статистика по продавцу.
- Привязка продавца к категории.

### Cash

- Снятия наличных.
- Ручные наличные расходы.
- Приблизительный баланс.

### Settings

- Пользователи семьи.
- Безопасность.
- Настройки импорта.
- Язык интерфейса.
- Лимит крупной операции, по умолчанию 5000 UAH.
- Бюджеты и лимиты по категориям.

### Notifications

- Уведомления внутри веб-приложения.
- Операции, требующие проверки после импорта.
- Предупреждения о приближении к лимитам.
- Превышение лимитов.

## Визуальный стиль

Сервис должен ощущаться как спокойный рабочий инструмент:

- плотная, но аккуратная информационная структура;
- фокус на таблицах, фильтрах, графиках;
- без маркетинговой landing page на первом экране;
- понятные действия и состояния.
