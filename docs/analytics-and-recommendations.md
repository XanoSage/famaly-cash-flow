# Analytics And Recommendations

## Аналитика MVP

### Сводка за период

- Доходы.
- Расходы.
- Остаток.
- Средний расход в день.

### Расходы по категориям

- Категории.
- Подкатегории.
- Доля от общих расходов.
- Абсолютные суммы.

### Динамика

- Расходы по дням.
- Расходы по неделям.
- Расходы по месяцам.

### Продавцы

- Топ продавцов по сумме.
- Топ продавцов по количеству операций.
- Расходы в конкретном продавце за период.

### Накопления

- Накоплено за период.
- Динамика накоплений по дням или месяцам.
- Сравнение расходов и накоплений.
- Прогноз накоплений за год при текущем темпе.

### Бюджеты

- Общий месячный семейный бюджет.
- Лимиты по важным категориям.
- Процент использования бюджета.
- Предупреждения при приближении к лимиту.

### Подписки

- Возможные повторяющиеся платежи по продавцу и похожей сумме.
- Сумма подписок за период.
- Инсайты по росту или изменению регулярных платежей.

## Рекомендации MVP

В первой версии рекомендации должны быть простыми и фактическими:

- "На еду ушло 42% расходов".
- "Топ-3 продавца: ATB, Silpo, Apteka".
- "Средний расход в день: 1240 UAH".
- "Самая крупная категория за период: Еда".
- "За период накоплено 850 UAH через округления и 1%".
- "Если темп сохранится, за год накопится примерно 10200 UAH".

## Будущие рекомендации

- Сравнение с прошлым периодом.
- Предупреждения о перерасходе.
- Поиск аномалий.
- Рекомендации по экономии.
- AI-отчет человеческим языком.
- Экран подписок.
- Telegram-уведомления.

## Требования к рекомендациям

- Не делать слишком смелых выводов без данных.
- Отделять факты от предположений.
- Показывать, на каких данных основан вывод.
- Давать рекомендации коротко и понятно.
